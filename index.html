Partage entre appareils (optionnel)

Par défaut, tes données restent dans ce navigateur. Pour que toi et ton conjoint voyiez les mêmes plats, quizz et menu en direct sur vos appareils — protégés par un mot de passe — connecte l'app à une base de données gratuite (Firebase). À faire une fois, par l'un de vous deux :

  1. Va sur console.firebase.google.com, connecte-toi avec un compte Google (gratuit) et crée un projet.
  2. Dans le menu de gauche : Créer une base de données → Realtime Database → démarre en mode test (on protège ça juste après).
  3. Menu de gauche → Authentication → Get started → onglet Sign-in method → active E-mail/Mot de passe.
  4. Toujours dans Authentication, onglet Users → Add user → crée un e-mail + mot de passe (le même que ton conjoint utilisera pour se connecter).
  5. Retourne dans Realtime Database → onglet Règles : remplace le contenu par {"rules": {".read": "auth != null", ".write": "auth != null"}} puis Publier. (Cela remplace le mode test ouvert par une vraie protection par mot de passe.)
  6. Paramètres du projet (roue crantée, en haut à gauche) → section Vos applications → ajoute une appli Web (icône </>) → copie l'objet firebaseConfig affiché.
  7. Colle-le ci-dessous avec un code du foyer (juste pour séparer vos données des autres utilisateurs Firebase, pas une protection en soi) et l'e-mail/mot de passe créés à l'étape 4. Fais pareil sur l'appareil de ton conjoint.

⚪ Non connecté (données locales uniquement sur cet appareil)

🔒 Avec les règles ci-dessus, seul quelqu'un connecté avec cet e-mail/mot de passe peut lire ou modifier les données. Le mot de passe n'est pas sauvegardé par l'app — Firebase garde juste ta session ouverte sur cet appareil (comme un site web classique), jusqu'à ce que tu te déconnectes.

Intervention ponctuelle de Claude

Si un jour tu veux que Claude complète des recettes ou touche au menu directement dans tes données partagées, connecte-toi ci-dessus puis clique ce bouton : il génère un accès temporaire (~1h, pas ton mot de passe) à coller dans une conversation. Rien n'est donné en continu — à chaque fois, c'est toi qui choisis de le partager.

Sans connexion (onglet Partage), tes données restent sauvegardées localement dans ce navigateur.
